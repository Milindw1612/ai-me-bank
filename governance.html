Risk & Governance Framework

Governance — Human Control, Data Protection & Regulatory Alignment

How Ai-ME BANK's 3 Agentic AI use cases stay accountable: what's automated vs. what always requires a human decision, how PII is protected, and where RBI regulatory considerations apply.

MSME Credit CAM Prep

Read-only bureau/AA integration. Every recommendation is a draft — a human credit officer holds final lending authority on every application, no exceptions.

Fraud Detection

Auto-freeze only fires above a governance-approved confidence threshold. Everything in the mid-confidence band routes to a human analyst before any account action.

Collections Recovery

Negotiation is bounded by pre-approved settlement bands per DPD bucket. Anything outside the band, or any customer pushback, escalates to a human officer.

Zero-tolerance rule: customer name, PAN, Aadhaar, account/card number, and outstanding balances never enter an LLM prompt in raw form — verified in code via the shared PII middleware (see shared/middleware.py).
What's Protected
  • Customer name, PAN, Aadhaar — tokenised (HMAC) at the API boundary, before any agent sees it
  • Account number, card number, IFSC — encrypted (AES-256-GCM) at rest, tokenised in transit between agents
  • Outstanding balance, loan amount, salary — scrubbed from any payload before it reaches an LLM prompt
  • Device ID, phone number — tokenised; only decrypted at the point of actually sending a notification

Account Aggregator Framework

MSME Credit CAM Prep's bank-feed access is consent-based via the RBI-regulated Account Aggregator framework — the bank never pulls data outside an explicit, time-bound, revocable customer consent, and the AA integration is read-only.

Digital Lending Guidelines

MSME Credit CAM Prep's pricing recommendation is disclosed to the applicant alongside the human-approved sanction, consistent with RBI's digital lending disclosure expectations — the AI's role in preparing the recommendation is not hidden from the borrower.

Fair Practices Code

Collections Recovery's automated outreach (tone, frequency, permitted hours) is bounded by the same Fair Practices Code rules that apply to human collections staff — the Escalation Agent's job is specifically to catch any interaction drifting outside those norms and hand it to a person.

IT Outsourcing & Model Risk

All 3 use cases keep model calls (Anthropic Claude API) and infrastructure (AWS) documented as part of the bank's IT outsourcing register, with Langfuse providing the audit trail RBI's IT governance expectations would look for — every model call, cost, and latency is traceable.
