15
pgvector Schema — Credit Policy RAG Tables
Predecessor: Step 14 Successor: Step 16 Owner: AI/ML Engineer Time: 45 minutes 
CREATE TABLE credit_policy_chunks (
  id SERIAL PRIMARY KEY,
  chunk_text TEXT NOT NULL,
  source_ref VARCHAR(200) NOT NULL,
  source_type VARCHAR(50) NOT NULL,       -- rbi_master_direction / internal_credit_policy / kyc_aml_guideline
  embedding VECTOR(1536),
  chunk_tsv TSVECTOR,
  created_at TIMESTAMP DEFAULT now()
);

-- Dense vector index (HNSW, cosine distance)
CREATE INDEX credit_policy_hnsw_idx ON credit_policy_chunks
  USING hnsw (embedding vector_cosine_ops) WITH (m = 16, ef_construction = 64);

-- Sparse text index (for hybrid search)
CREATE INDEX credit_policy_tsv_idx ON credit_policy_chunks USING gin(chunk_tsv);

-- Auto-update tsvector on insert/update
CREATE FUNCTION credit_policy_tsv_trigger() RETURNS trigger AS $$
BEGIN
  NEW.chunk_tsv := to_tsvector('english', NEW.chunk_text);
  RETURN NEW;
END $$ LANGUAGE plpgsql;
CREATE TRIGGER tsvectorupdate BEFORE INSERT OR UPDATE ON credit_policy_chunks
  FOR EACH ROW EXECUTE FUNCTION credit_policy_tsv_trigger();

This is the table shared/rag.py's hybrid_search() queries — see shared/rag.py in the source repo.

19
Credit & Compliance Policy Ingestion
Predecessor: Step 18 Successor: Step 20 Owner: AI/ML Engineer Time: 1 week (incl. document collection)

Ingest the policy documents collected in Step 0: RBI Master Direction on MSME lending, internal credit policy manual, KYC/AML guidelines, Digital Lending Guidelines, Fair Practices Code.

Ingestion Pipeline
1. Upload PDFs to s3://ai-me-bank-cam-docs/policy-source/
2. Extract text (pdfplumber), chunk at 400 tokens / 80 token overlap
3. For each chunk: embed_query(chunk_text) → insert into credit_policy_chunks
   with source_ref (e.g. "RBI/2022-23/Master Direction MSME, para 4.2")
   and source_type (rbi_master_direction / internal_credit_policy / kyc_aml_guideline)
⚠ Credit Risk Analyst must review chunk boundaries for any policy paragraph that defines a hard numeric limit (e.g. loan-to-value caps) — a chunk split can silently separate a rule from its threshold value.

Hybrid RAG System
Predecessor: Step 19 Successor: Step 21 Owner: AI/ML Engineer Time: 2 days
shared/rag.py — already in the repo
FunctionWhat it does
embed_query()Bedrock Titan V2 embedding for the incoming question
hybrid_search()Dense (HNSW cosine) + sparse (tsvector) search run in parallel, combined via Reciprocal Rank Fusion
policy_qa()Retrieves top chunks, asks Claude Sonnet to answer grounded only in that context, returns answer + source refs

The MSME Risk & Cash-Flow Agent (a1_msme_credit/risk_cashflow_agent.py) calls policy_qa() before scoring — see Architecture for how this fits the shared platform.

21
RAG Validation Test Suite
Predecessor: Step 20 Successor: Step 22 Owner: AI/ML Engineer Time: 3 days

20 real credit/compliance Q&As run via test_rag.py — must score ≥ 18/20 before proceeding. Credit Risk Analyst reviews every answer against actual policy text.

Example Validation Questions
Q: "What is the maximum loan-to-value ratio for an unsecured MSME working capital loan?"
Q: "Does a first-time applicant with 18 months business vintage qualify under our credit policy?"
Q: "What KYC documents are mandatory for a proprietorship MSME applicant?"
⚠ Gate 1 (see RACI): this score must reach ≥ 18/20 before app deployment (Part 3) begins. If below threshold, re-chunk, re-ingest, re-test — do not proceed on a failing score.
