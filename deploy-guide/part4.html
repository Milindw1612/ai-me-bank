Core Banking System (CBS) OAuth2 Integration
Predecessor: Step 34 Successor: Step 36 Owner: Backend Engineer Time: Week 1 → 6 (vendor-bound, HIGH risk)

The single highest-risk external dependency (see RACI). Feeds every use case's account lookup and gated write actions (freeze/clear, PTP recording).

# OAuth2 client-credentials grant against CBS sandbox
curl -X POST $CBS_BASE_URL/oauth/token \
  -d "grant_type=client_credentials&client_id=$CBS_CLIENT_ID&client_secret=$CBS_CLIENT_SECRET"

# Test read-only account lookup
curl -X GET $CBS_BASE_URL/accounts/lookup -H "Authorization: Bearer $token" -d 'account_token=...'

# Test gated write action (fraud freeze) — SANDBOX ONLY until Step 45 governance sign-off
curl -X POST $CBS_BASE_URL/transactions/$TXN_ID/freeze -H "Authorization: Bearer $token"
⛔ Do not point any write-action test at a production CBS account until Gate 3 (PII sign-off) and the Step 45 go-live checklist are both complete. Freeze/clear actions are irreversible without a manual CBS ops ticket.

PII Security

38
PII Boundary Map Review
Predecessor: Step 37 Successor: Step 39 Owner: Security Engineer Time: 3 days

Written sign-off: every PII field mapped to its lifecycle (tokenise / encrypt / decrypt boundary), reviewed against shared/middleware.py's _PII_FIELDS set.

PII Fields & Their Boundary

customer_name, account_number
Tokenised (HMAC) at API entry — never reaches an agent in raw form
pan, aadhaar, card_number
Encrypted (AES-256-GCM) at rest; decrypted only at the exact point of external API call
outstanding_balance, loan_amount
Scrubbed by safe_json() before any LLM prompt — visible to humans via the Officer Portal only
phone, mobile
Tokenised; decrypted only inside whatsapp_send_reminder() / sms_send_reminder() at send time
