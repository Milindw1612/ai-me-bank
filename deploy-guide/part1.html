AWS Accounts

1
Create Two Separate AWS Accounts (Prod + Staging)
Predecessor: Step 0 (prerequisites collected) Successor: Step 2 Owner: DevOps Engineer Time: 2–4 hours

Separate AWS accounts for prod and staging is not optional. It prevents accidental prod deployments and allows billing separation.

Action — Create Staging Account
1. Go to https://aws.amazon.com → Create an AWS Account
2. Email:   ai-me-bank-staging@yourbank.com
3. Account name: ai-me-bank-staging
4. Choose "Business" account
5. Add billing details (corporate card)
6. Select Support Plan: Business (recommended for production systems)
7. Note down Account ID (12-digit number shown after creation)
Action — Create Prod Account
1. Open new browser incognito window
2. Go to https://aws.amazon.com → Create an AWS Account
3. Email:   ai-me-bank-prod@yourbank.com
4. Account name: ai-me-bank-prod
5. Same billing details
6. Note down Account ID
Choose a Region — Use This Throughout
# Recommended: ap-south-1 (Mumbai) — data residency, lowest latency to core banking DC
export AWS_REGION=ap-south-1
export STAGING_ACCOUNT_ID=xxxxxxxxxxxx
export PROD_ACCOUNT_ID=yyyyyyyyyyyy
✓ After this step you have two isolated AWS accounts. All subsequent steps are performed in BOTH accounts (staging first, then prod).
2
Secure Root Account — MFA + Billing Alerts
Predecessor: Step 1 Successor: Step 3 Owner: DevOps Engineer Time: 30 minutes per account

Do this in BOTH accounts immediately after creation. Root account compromise is irreversible.

  • Sign in to AWS Console as root → My Account → Multi-Factor Authentication (MFA) → Activate MFA → Virtual MFA device → scan QR with Authy/Google Authenticator
  • Billing → Billing Preferences → Enable PDF invoices, Free Tier alerts, CloudWatch billing alerts
  • Create a billing alert: CloudWatch → Alarms → Create Alarm → Billing → EstimatedCharges → threshold ₹50,000 → SNS email alert to DevOps lead
⚠ Never use root account for daily operations. After Step 3, lock the root credentials in a password manager and only use the IAM admin user.
3
IAM Admin User + Least-Privilege Service Roles
Predecessor: Step 2 Successor: Step 4 Owner: DevOps + Security Engineer Time: 1 hour

Do in both accounts. Create one admin user for DevOps, then least-privilege service roles for the application.

Console: IAM → Users → Create User
Username: ai-me-bank-devops
Access type: ✓ Programmatic access + ✓ AWS Console access
Attach policy: AdministratorAccess (for DevOps lead only)
Enable MFA on this user too
Create Application Service Roles (IAM → Roles)
# Role 1: EKS Node Role
Name: ai-me-bank-eks-node-role
Trusted entity: EC2
Policies:
  - AmazonEKSWorkerNodePolicy
  - AmazonEC2ContainerRegistryReadOnly
  - AmazonEKS_CNI_Policy
  - AmazonSSMManagedInstanceCore

# Role 2: Application Pod Role (IRSA)
Name: ai-me-bank-app-role
Trusted entity: Web identity (EKS OIDC)
Custom policy (inline):
{
  "Version": "2012-10-17",
  "Statement": [
    {"Effect":"Allow","Action":["s3:GetObject","s3:PutObject","s3:ListBucket"],
     "Resource":["arn:aws:s3:::ai-me-bank-cam-docs/*","arn:aws:s3:::ai-me-bank-cam-docs"]},
    {"Effect":"Allow","Action":["secretsmanager:GetSecretValue"],
     "Resource":"arn:aws:secretsmanager:ap-south-1:*:secret:ai-me-bank-*"},
    {"Effect":"Allow","Action":["kms:Decrypt","kms:GenerateDataKey"],"Resource":"*"}
  ]
}

# Role 3: RDS Enhanced Monitoring
Name: ai-me-bank-rds-monitoring
Trusted entity: monitoring.rds.amazonaws.com
Policy: AmazonRDSEnhancedMonitoringRole
4
Configure AWS CLI with Named Profiles
Predecessor: Step 3 (IAM user access keys created) Successor: Step 5 Owner: DevOps Engineer (local machine) Time: 20 minutes
Terminal — Configure Profiles
# Configure staging profile
aws configure --profile ai-me-bank-staging
# AWS Access Key ID: [paste from IAM user creation]
# AWS Secret Access Key: [paste secret]
# Default region: ap-south-1
# Default output: json

# Configure prod profile
aws configure --profile ai-me-bank-prod
# Same as above with prod account credentials

# Verify both profiles work
aws sts get-caller-identity --profile ai-me-bank-staging
aws sts get-caller-identity --profile ai-me-bank-prod
Set Environment Variables (add to ~/.bashrc or ~/.zshrc)
export AWS_PROFILE=ai-me-bank-staging   # default to staging
export AWS_REGION=ap-south-1
export PROJECT=ai-me-bank
export ENV=staging    # change to prod when deploying prod
✓ From this point, all CLI commands use --profile ai-me-bank-staging or --profile ai-me-bank-prod. Always double-check your profile before running any command.

Networking

5
Create VPC, Subnets, Route Tables
Predecessor: Step 4 Successor: Step 6 Owner: DevOps Engineer Time: 45 minutes

Create this network layout in BOTH accounts. Public subnets (ALB), private subnets (EKS pods, RDS).

AWS CLI — Reproducible
# Create VPC
VPC_ID=$(aws ec2 create-vpc --cidr-block 10.0.0.0/16 \
  --profile ai-me-bank-staging --query 'Vpc.VpcId' --output text)
aws ec2 create-tags --resources $VPC_ID \
  --tags Key=Name,Value=ai-me-bank-vpc Key=Project,Value=ai-me-bank \
  --profile ai-me-bank-staging

aws ec2 modify-vpc-attribute --vpc-id $VPC_ID --enable-dns-hostnames --profile ai-me-bank-staging
aws ec2 modify-vpc-attribute --vpc-id $VPC_ID --enable-dns-support --profile ai-me-bank-staging

# Public Subnet AZ-a (for ALB)
PUB_A=$(aws ec2 create-subnet --vpc-id $VPC_ID \
  --cidr-block 10.0.1.0/24 --availability-zone ap-south-1a \
  --profile ai-me-bank-staging --query 'Subnet.SubnetId' --output text)
aws ec2 create-tags --resources $PUB_A \
  --tags Key=Name,Value=ai-me-bank-public-1a Key=kubernetes.io/role/elb,Value=1 \
  --profile ai-me-bank-staging

# Public Subnet AZ-b
PUB_B=$(aws ec2 create-subnet --vpc-id $VPC_ID \
  --cidr-block 10.0.2.0/24 --availability-zone ap-south-1b \
  --profile ai-me-bank-staging --query 'Subnet.SubnetId' --output text)

# Private Subnet AZ-a (EKS pods)
PRIV_A=$(aws ec2 create-subnet --vpc-id $VPC_ID \
  --cidr-block 10.0.11.0/24 --availability-zone ap-south-1a \
  --profile ai-me-bank-staging --query 'Subnet.SubnetId' --output text)
aws ec2 create-tags --resources $PRIV_A \
  --tags Key=Name,Value=ai-me-bank-private-1a Key=kubernetes.io/role/internal-elb,Value=1 \
  --profile ai-me-bank-staging

# Private Subnet AZ-b (RDS standby)
PRIV_B=$(aws ec2 create-subnet --vpc-id $VPC_ID \
  --cidr-block 10.0.12.0/24 --availability-zone ap-south-1b \
  --profile ai-me-bank-staging --query 'Subnet.SubnetId' --output text)

echo "VPC: $VPC_ID"
echo "Public:  $PUB_A  $PUB_B"
echo "Private: $PRIV_A $PRIV_B"
# Save these values — you'll need them in every subsequent step
6
Security Groups
Predecessor: Step 5 Successor: Step 7 Owner: DevOps + Security Engineer Time: 30 minutes
Create 4 Security Groups
# ALB SG — public 443 only
aws ec2 create-security-group --group-name ai-me-bank-alb-sg \
  --description "ALB - public HTTPS" --vpc-id $VPC_ID --profile ai-me-bank-staging
aws ec2 authorize-security-group-ingress --group-id $ALB_SG \
  --protocol tcp --port 443 --cidr 0.0.0.0/0 --profile ai-me-bank-staging

# API SG — only from ALB, port 8000
aws ec2 create-security-group --group-name ai-me-bank-api-sg \
  --description "API pods - from ALB only" --vpc-id $VPC_ID --profile ai-me-bank-staging
aws ec2 authorize-security-group-ingress --group-id $API_SG \
  --protocol tcp --port 8000 --source-group $ALB_SG --profile ai-me-bank-staging

# RDS SG — only from API, port 5432
# Redis SG — only from API, port 6379
💡 No security group should ever allow inbound from 0.0.0.0/0 except the ALB SG on port 443. This is checked in Step 38's PII boundary review.
7
NAT Gateway & Internet Gateway
Predecessor: Step 6 Successor: Step 8 Owner: DevOps Engineer Time: 20 minutes
Internet Gateway + Route Tables
IGW=$(aws ec2 create-internet-gateway --profile ai-me-bank-staging \
  --query 'InternetGateway.InternetGatewayId' --output text)
aws ec2 attach-internet-gateway --internet-gateway-id $IGW --vpc-id $VPC_ID \
  --profile ai-me-bank-staging

# Public route table → IGW
PUB_RT=$(aws ec2 create-route-table --vpc-id $VPC_ID --profile ai-me-bank-staging \
  --query 'RouteTable.RouteTableId' --output text)
aws ec2 create-route --route-table-id $PUB_RT \
  --destination-cidr-block 0.0.0.0/0 --gateway-id $IGW --profile ai-me-bank-staging

# NAT Gateway (1 per AZ for staging cost savings, 2 for prod HA)
EIP=$(aws ec2 allocate-address --domain vpc --profile ai-me-bank-staging --query 'AllocationId' --output text)
NAT=$(aws ec2 create-nat-gateway --subnet-id $PUB_A --allocation-id $EIP \
  --profile ai-me-bank-staging --query 'NatGateway.NatGatewayId' --output text)

Secrets & Keys

8
AWS Secrets Manager — Seed All Secrets
Predecessor: Step 7 Successor: Step 9 Owner: DevOps + Backend Engineer Time: 45 minutes
Seed Every External Credential
aws secretsmanager create-secret --name ai-me-bank-anthropic-key --secret-string "sk-ant-..." --profile ai-me-bank-staging
aws secretsmanager create-secret --name ai-me-bank-cbs-client-secret --secret-string "..." --profile ai-me-bank-staging
aws secretsmanager create-secret --name ai-me-bank-aa-client-secret --secret-string "..." --profile ai-me-bank-staging
aws secretsmanager create-secret --name ai-me-bank-bureau-api-key --secret-string "..." --profile ai-me-bank-staging
aws secretsmanager create-secret --name ai-me-bank-aml-api-key --secret-string "..." --profile ai-me-bank-staging
aws secretsmanager create-secret --name ai-me-bank-whatsapp-token --secret-string "..." --profile ai-me-bank-staging
aws secretsmanager create-secret --name ai-me-bank-pii-encryption-key --secret-string "$(openssl rand -hex 32)" --profile ai-me-bank-staging
aws secretsmanager create-secret --name ai-me-bank-langfuse-keys --secret-string '{"public":"pk-lf-...","secret":"sk-lf-..."}' --profile ai-me-bank-staging
⚠ The PII encryption key (Step 8) is the single most sensitive value in the entire system — every tokenised customer field depends on it. Rotate annually; never log it; never commit it.
9
KMS Encryption Keys
Predecessor: Step 8 Successor: Step 10 Owner: Security Engineer Time: 30 minutes
Create 3 Separate KMS Keys
aws kms create-key --description "ai-me-bank RDS encryption" --profile ai-me-bank-staging
aws kms create-key --description "ai-me-bank S3 audit exports" --profile ai-me-bank-staging
aws kms create-key --description "ai-me-bank S3 CAM/KYC documents" --profile ai-me-bank-staging

# Enable annual auto-rotation on all 3
aws kms enable-key-rotation --key-id $KMS_KEY_RDS --profile ai-me-bank-staging
✓ Separate keys per data class means a compromise of one key doesn't expose the others — audit logs stay recoverable even if the CAM-document key were ever rotated out.

Container Platform

10
ECR Repository + Image Build
Predecessor: Step 9 Successor: Step 11 Owner: DevOps Engineer Time: 1 hour 
aws ecr create-repository --repository-name ai-me-bank-api --profile ai-me-bank-staging

# Build and push (from agentic-ai/ repo root)
cd agentic-ai

11
EKS Cluster Provision
Predecessor: Step 10 Successor: Step 12 Owner: DevOps Engineer Time: 30–45 minutes (cluster provisioning) 
eksctl create cluster \
  --name ai-me-bank-eks \
  --region ap-south-1 \
  --version 1.30 \
  --vpc-private-subnets=$PRIV_A,$PRIV_B \
  --vpc-public-subnets=$PUB_A,$PUB_B \
  --nodegroup-name ai-me-bank-nodes \
  --node-type t3.large \
  --nodes 3 \
  --nodes-min 2 \
  --nodes-max 6 \
  --managed \
  --with-oidc \
  --profile ai-me-bank-staging

aws eks update-kubeconfig --name ai-me-bank-eks --region ap-south-1 --profile ai-me-bank-staging
kubectl get nodes
✓ 3 nodes Ready confirms the cluster is live. IRSA (--with-oidc) is required for Step 3's Application Pod Role to work.
12
WAF v2 + ACM SSL Certificate
Predecessor: Step 11 Successor: Step 13 (Part 2) Owner: Security Engineer + DevOps Time: 45 minutes 
# WAF ACL with AWS managed rules + rate limiting
aws wafv2 create-web-acl --name ai-me-bank-waf --scope REGIONAL \
  --default-action Allow={} \
  --rules file://waf-rules.json \
  --profile ai-me-bank-staging

# ACM SSL cert (DNS-validated)
aws acm request-certificate --domain-name ai-me-bank.yourdomain.com \
  --validation-method DNS --profile ai-me-bank-staging
💡 Test the WAF with OWASP payloads before Step 29 (ALB Ingress) wires it in — a misconfigured rate limit here blocks legitimate credit officer traffic in production.
