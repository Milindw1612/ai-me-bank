Technical Design

Shared Platform Architecture

All 3 use cases run on one shared agent platform — built once, reused across MSME Credit CAM Prep, Fraud Detection, and Collections Recovery, so each incremental use case costs a fraction of the first.

Platform Components
☁️ Infrastructure
Dedicated AI environment, isolated from core banking production
COMPUTE
AWS EKS (Kubernetes) — autoscaling agent pods per use case
DATABASE
AWS RDS PostgreSQL + pgvector, per-use-case audit schema
SECRETS
AWS Secrets Manager — CBS/bureau/AA credentials, never in code
🤖 Agent Orchestration
LangGraph state machines with human-in-loop gates at every decision
FRAMEWORK
LangGraph — fan-out/fan-in sub-agents, PostgreSQL checkpointing
LLM ROUTING
Sonnet 4.5 for risk reasoning; Haiku 4.5 for scoring/classification
OBSERVE
Langfuse — every LLM call traced: tokens, latency, cost, errors
🔗 Integration Map
Read-only wherever possible — no automation holds unilateral write authority on money movement
BUREAU
CIBIL score pull, Account Aggregator consent-based bank feeds
TAX / GST
GSTR-1/3B filing verification for MSME applicants
CHANNELS
WhatsApp, SMS, IVR — shared outreach layer for collections
🛡️ Security & Compliance
Non-negotiable design rules applied across all 3 use cases
GATEWAY
Single common API gateway + OAuth2/mTLS authentication in front of all 3 use cases — see Helicopter View
PII POLICY
Customer name, PAN, Aadhaar, account/card number never enter an LLM prompt
ENCRYPT
AES-256 at rest (KMS), TLS 1.3 in transit
HUMAN GATE
Credit sign-off, confirmed-fraud calls, and settlement outcomes stay human
